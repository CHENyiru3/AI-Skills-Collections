# Maintenance Loop

Use this loop for most wiki tasks.

## Build or restructure

1. Inspect the schema and current folder taxonomy.
2. Propose the durable page families or moves.
3. Create or move pages.
4. Update affected links and navigation files.
5. Verify there are no stale paths or broken wikilinks.

## Ingest

1. Read the new source material.
2. Identify durable subjects touched by the source.
3. Create new standalone pages only when they pass the vault's threshold.
4. Update canonical pages and cross-links.
5. Update `index.md`, `log.md`, and `inbox.md` as required.
6. Verify link integrity and page placement.

## Query

1. Start with map and index pages.
2. Read canonical pages before drilling into source detail.
3. Answer with clear grounding.
4. File reusable syntheses back into the wiki when they have lasting value.

## Lint

Check for:

- orphan pages
- shallow stubs
- duplicate subjects
- stale claims
- missing references
- missing navigation updates
- files placed in the wrong family or subfolder
- schema drift between `AGENTS.md` and actual content
