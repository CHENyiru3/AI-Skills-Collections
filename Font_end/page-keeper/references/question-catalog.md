# Question Catalog

Ask only the questions that remain unresolved after repo inspection.

## Section governance

Use when it is unclear what may change:
- Which sections can future agents polish without asking first?
- Which sections should only change when you give exact wording?
- Which sections must never be rewritten unless you explicitly request it?

## Style and tone

Use when tone is not obvious from the repo:
- Should the homepage voice stay formal, slightly personal, or fully professional?
- Can future agents tighten wording and structure, or should they preserve the existing phrasing closely?
- Are there sections where a more human or casual tone is preferred?

## Publications and references

Use when multiple valid layouts exist:
- How should publications be ordered: by year, status, featured-first, or another rule?
- Should accepted/forthcoming manuscripts appear before in-preparation items?
- Are conferences, posters, and presentations separate from manuscripts?
- Are author symbols and notes fixed?

## CV and versioning

Use when assets or dates are derived:
- Should the CV version always be taken from the filename?
- Are downloadable asset names part of a stable convention?
- Should agents rename files when updating dates, or only update references?

## Identity and factual sensitivity

Use when facts are sensitive or future-facing:
- Which identity statements require your exact wording?
- Should affiliations, positions, or future plans only be updated from direct user instruction?
- Are institution names allowed to be shortened?

## Layout and structure

Use when refactoring is plausible:
- May future agents centralize duplicated content into shared includes?
- Should page structure remain fixed even if the repo would benefit from reuse?
- Are inline styles acceptable to preserve, or should future agents consolidate them?
