# Python Skills Catalog

This document catalogs the Python library development skills available in this repository.

## Skills Overview

| Skill Directory | Skill Name | Focus |
|----------------|------------|-------|
| `project-setup` | `setting-up-python-libraries` | Project scaffolding, `pyproject.toml`, CI, tooling |
| `code-quality` | `improving-python-code-quality` | Ruff, mypy, refactoring, Pythonic patterns |
| `testing-strategy` | `testing-python-libraries` | Pytest, fixtures, coverage, Hypothesis |
| `security-audit` | `auditing-python-security` | Bandit, pip-audit, Semgrep, secrets scanning |
| `api-design` | `designing-python-apis` | API ergonomics, deprecation, compatibility |
| `documentation` | `documenting-python-libraries` | Docstrings, Sphinx, tutorials, API docs |
| `packaging` | `packaging-python-libraries` | Builds, wheels, PyPI, trusted publishing |
| `release-management` | `managing-python-releases` | SemVer, changelogs, release automation |
| `performance` | `optimizing-python-performance` | Profiling, memory analysis, benchmarking |
| `cli-development` | `building-python-clis` | Click, Typer, entry points, CLI testing |
| `community` | `building-python-communities` | CONTRIBUTING, templates, governance |
| `library-review` | `reviewing-python-libraries` | Holistic audits across quality dimensions |

## Bundles

- Foundations: `project-setup`, `code-quality`, `testing-strategy`
- Distribution: `packaging`, `release-management`, `cli-development`
- Quality: `security-audit`, `performance`, `api-design`, `library-review`
- Docs and Community: `documentation`, `community`

## Install

Copy leaf skill directories into either `~/.codex/skills/` or `~/.claude/skills/`, or enable the repo through `.claude-plugin/marketplace.json` for Claude Code.
