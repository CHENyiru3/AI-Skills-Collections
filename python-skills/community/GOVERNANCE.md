# Governance Guide

Use lightweight governance until the project size demands more process.

## Roles

- Maintainer: merges changes, makes release decisions, owns roadmap tradeoffs.
- Reviewer: provides technical review and helps triage.
- Contributor: opens issues and pull requests.

## Operating Rules

- Record decisions in issues, PRs, or docs instead of private chats.
- Prefer one clear maintainer decision over long unresolved discussion.
- Mark breaking changes and roadmap decisions explicitly.

## Triage Cadence

- New issues: acknowledge quickly.
- Reproduced bugs: label and prioritize.
- Stale proposals: close with context rather than letting them drift forever.

## Access

- Grant elevated repo permissions only to people who are already doing the work.
- Separate release permissions from broad admin rights when possible.
