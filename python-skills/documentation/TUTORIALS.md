# Tutorial Writing Guide

Tutorials should help a new user finish one meaningful task end to end.

## Structure

1. State the outcome.
2. List prerequisites.
3. Give a copy-pasteable quick start.
4. Explain the result and next steps.

## Writing Rules

- Optimize for first success, not completeness.
- Use one coherent example instead of many partial fragments.
- Keep the user moving; long theory belongs in concept docs, not tutorials.
- Show expected output so readers know they are on track.

## Good Tutorial Topics

- Install and run the first command
- Load data and perform one core operation
- Configure one realistic workflow
